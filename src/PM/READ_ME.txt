1.c - генерирует файлы в директории ./data
использовалась для дальнейшей проверки работоспособности других программ
---------
all_in_one.c - объединяет содержимое файлов директории ./data в файл wad.txt,
уничтожая файлы директории ./data, если wad.txt же существует, то дополняется
без потери данных, в начале wad.txt стоит строка с количеством строк данных
---------
ask2wad.c - модификация предыдущей программы, созданная по просьбе Ильи,
предполагающая что в файл wad.txt будут собираться данные только из файлов,
адреса которых записаны в файл indexTime.txt
---------
one_to_bmp_xy.c - считывает данные из wad.txt, высчитывает позицию
точек траектории на картинке и записывает в файл bmp_xy.txt строки
следующего содержания:

	*время_от_начала_дня_в_секундах* *x_точки_на_картинке* *y_точки_на_картинке*

все данные файла bmp_xy.txt - int

принцип работы заключается в определении расстояния точки от края картинки,
как расстояния от точки до прямой на плоскости вида ax+by=c, соответственно
нужен файл coor.txt вида:

	*ширина* *высота*
	*x_gps_левой_верхней_точки* *y_gps_левой_верхней_точки*
	*x_gps_правой_верхней_точки* *y_gps_правой_верхней_точки*
	*x_gps_левой_нижней_точки* *y_gps_левой_нижней_точки*
	*x_gps_правой_нижней_точки* *y_gps_правой_нижней_точки*

эти данные перерабатываются и записываются в файл ABC.txt, все это нужно
просто чтобы итоговая формула расстояния была читабельной,
если любой из параметров coor.txt изменится, просто удалить ABC.txt,
программа создаст новый.
---------
test.c - просто игрушка, рисующая в файле test.bmp по командам,
записанным в файле pic.txt
	поддерживает команды:
1) line (отрезок, заданный двумя точками)
2) circle (окружность настраиваемой толщины)
3) fill (рекурсивно-работающая заливка)
4) square (прямоугольник, заданный двумя точками)

пример pic.txt прилагается (буква C отрисовке не поддается)
